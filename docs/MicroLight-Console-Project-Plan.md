# MicroLight Console 项目计划文档

## 1. 文档目的

本文档用于记录 `MicroLight Console` 的实施计划、阶段目标、任务拆解、依赖关系、里程碑和进度状态。

它服务于两个目标：

1. 把设计方案拆成可执行的开发步骤
2. 在项目推进过程中持续记录完成情况与下一步安排

关联设计文档：

- [MicroLight-Console-Design.md](D:\Aerlox\work\MicroLight Console\docs\MicroLight-Console-Design.md)

## 2. 项目目标

`MicroLight Console` 的目标是构建一个基于 `Electron + Node.js + Vue 3` 的本地微服务桌面控制台，用更低资源占用替代 IntelliJ IDEA 在“微服务构建、启动、查看日志、基础监测”场景下的使用。

首期项目范围聚焦：

- Maven 项目扫描
- Spring Boot 3.x 启动类识别
- 单服务构建与启动
- 服务停止与重启
- 实时日志查看
- 基础状态监测
- Windows `exe` 安装包发布

## 3. 计划原则

- 先打基础，再做增强功能
- 先保证“能稳定跑通”，再追求“更快、更聪明”
- 先完成单服务主链路，再做服务组编排
- 先支持 Maven 3.x 稳定路径，再扩展 Maven 4 / `mvnd`
- 计划按阶段推进，每阶段都要有可验证交付物

## 4. 状态定义

- `未开始`：尚未进入执行
- `进行中`：已开始，正在推进
- `已完成`：已经完成并通过当前阶段验收
- `阻塞中`：存在外部依赖或关键问题，无法继续推进
- `已延期`：暂不处理，后移到后续阶段

## 9. 关键依赖与风险关注

关键依赖：

- Node.js 运行环境
- Electron 打包链路
- Java 17+
- Maven 3.x / 4.x 运行环境
- `mvnd` 可用性

关键风险：

- Maven 多模块复杂结构导致扫描异常
- 真实项目中 api / biz / app 混合模块命名导致服务展示和服务名归一化不稳定
- `mvnd` 与特定项目插件不兼容
- Electron 打包后路径、权限、杀毒软件影响
- 日志高吞吐导致渲染进程卡顿
